# 碰个橙子 · 个人网站

这是一个适合直接部署到 GitHub Pages 的静态个人网站。

## 现在已经有的内容

- 文艺风但不失专业感的单页主页
- 双身份展示：美食博主 / 算法与产品写作者
- 从 GitHub 自动拉取公开仓库并展示项目
- 写作、兴趣、技能、联系入口等区块
- 移动端适配和基础动效

## 最容易修改的地方

优先改 [script.js](./script.js) 里的 `siteProfile`：

- `heroFacts`：首屏右侧的小信息卡
- `skills`：专业技能标签
- `hobbies`：兴趣标签
- `links`：联系按钮
- `githubUsername`：如果以后换账号，改这里

如果你想补更完整的资料：

- 改 [index.html](./index.html) 里的正文文案
- 改 [styles.css](./styles.css) 里的颜色、字体、间距

## 如何本地预览

这是纯静态站，直接双击 `index.html` 就能打开。

如果你想用本地服务器预览，也可以在这个目录运行任意静态服务器，比如：

```powershell
npx serve .
```

## 如何部署到 GitHub Pages

1. 新建一个仓库。
2. 把当前目录下的文件上传到仓库根目录。
3. 在 GitHub 仓库设置里开启 Pages。
4. 发布源选择当前分支根目录。
5. 等待几分钟后，就会得到一个 `github.io` 链接。

如果你以后买了自己的域名，也可以继续把它绑定到 GitHub Pages。

## 下一步我建议你补的内容

1. 邮箱或更稳定的联系方式
2. 一份 PDF 简历
3. 2 到 4 篇你最想被看见的代表文章
4. 3 到 6 个最能代表你的项目说明

## 评论功能要不要加

现在先没加评论功能，因为评论意味着：

- 需要登录方案或第三方评论系统
- 需要审核和反垃圾处理
- 会增加维护成本

如果你后面想要评论，最适合 GitHub Pages 的方案通常是 `giscus` 这类基于 GitHub Discussions 的评论系统。
