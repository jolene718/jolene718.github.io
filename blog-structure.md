# 个人博客结构方案

这份文档不再讨论手写静态首页，而是直接基于你喜欢的那类个人博客来设计站点结构。

参考方向包括：

- `Hexo + Butterfly` 这类成熟博客主题
- 你提到喜欢的个人博客气质：
  - `leonis.cc`
  - `blog.orangex4.workers.dev`
  - `persvadisto.github.io`
- 你已经整理好的个人信息文档：[profile.md](/F:/blog/profile.md)

## 结论先说

你的网站不适合做成“单页作品集”。

更适合你的，是一个：

- 以博客为主骨架
- 以个人首页为门面
- 兼顾项目展示
- 能容纳双身份内容
- 可以慢慢长出友链、歌单、归档和碎片记录的站

也就是说，方向应该是：

`个人博客站` > `个人主页`

而不是：

`求职主页` > `博客附件`

## 从参考站里提炼出来的适合你的特征

### 1. 要有“真正博客”的结构感

你喜欢的那些站，不是一个长长的 landing page，而是有明确博客骨架的：

- 首页
- 分类
- 标签
- 归档
- About
- Links / 友链

这会天然让网站显得更“高技术力”和“真的在长期维护”，而不是一次性搭的展示页。

### 2. 首页应该像“最近在写什么、我是谁”的综合门面

适合你的首页不应该只是大标题 + 自我介绍，而应该同时承担：

- 让人快速知道你是谁
- 让人知道你最近在写什么
- 让人知道你有哪些长期栏目
- 让人知道你不只有职业面

### 3. 内容必须允许“专业”与“生活”并存

这是你和很多标准技术博客最不一样的地方。

你不是只写：

- 技术教程
- 学习笔记
- 开发记录

你同时还有：

- 美食记录
- 城市与日常观察
- 更个人化的感受表达

所以结构上不能把生活内容当成边角料，而要把它当成正式栏目。

### 4. 要有可持续扩展的页面

你后面很可能会继续加：

- 代表项目页
- 文章归档页
- 友链页
- 喜欢的歌单页
- 说说 / 碎片页

所以比起自己硬写一套 `html/css/js`，用现成博客框架和主题更合适。

## 推荐技术路线

### 推荐方案

- 博客框架：`Hexo`
- 主题：`Butterfly`

### 原因

- 主题成熟，适合个人博客
- 有明显的“博客感”
- 支持分类、标签、归档、友链、相册、说说等页面
- 视觉上比默认技术博客更轻盈，也更容易做出你想要的“文艺但不悬浮”的感觉
- 后面加歌单、评论、友链都方便

## 最适合你的站点结构

下面这套是针对你本人定制的，不是通用模板。

## 一、导航结构

### 顶部导航建议

1. 首页
2. 文章
3. 项目
4. 关于
5. 友链
6. 歌单

### 不建议放得太多的导航项

- 简历
- 联系我
- 技能树

这些可以放在 About 页或侧边栏，不需要顶到主导航里，不然会太像求职站。

## 二、首页结构

首页应该是“博客门面页”，不是“简历门面页”。

### 首页模块建议

1. 顶部欢迎区
2. 最近文章列表
3. 长期栏目入口
4. 项目精选
5. 双身份入口
6. 最近在听
7. 友链 / 想去的地方

### 1. 顶部欢迎区

#### 作用

- 第一眼立住你是谁
- 不要太商务
- 不要太像营销文案

#### 适合放的内容

- 站点名：`碰个橙子`
- 一句介绍：
  - “算法与产品并行，计算与思考同在。”
  - 或者更柔和一点：
  - “我把项目、写作、食物和生活放在同一个地方慢慢长大。”

#### 这里适合出现的元素

- 头像
- 一句话介绍
- GitHub / 小红书 / 邮箱入口
- 站点统计

### 2. 最近文章列表

这是首页最重要的主体之一。

应该优先展示你最近 / 最想被看到的文章，而不是把项目放在最前面。

#### 文章卡片适合的来源

- 专业号已有内容
- 公众号同源内容
- 未来独立博客文章

#### 第一批适合放在首页的代表文章方向

- 从产品视角拆透 SBTI
- AI 会产生意识吗？
- OpenClaw 使用体验 / 现象观察
- 和深度学习项目相关的一篇项目思考

### 3. 长期栏目入口

这一块比“技能标签”更重要，因为它能说明你的网站会继续长。

#### 推荐栏目

- `长思文章`
  - 算法、产品、AI、现象观察
- `项目手记`
  - GitHub 项目的背景、过程、复盘
- `食物与日常`
  - 美食、外卖测评、生活记录
- `说说 / 碎片`
  - 短句、临时想法、最近观察

### 4. 项目精选

首页里项目区不宜过大，但必须有。

#### 推荐方式

- 只精选 3 到 5 个项目
- 每个项目只写短摘要
- 详情放到项目页

#### 适合写法

- 项目名称
- 一句话项目说明
- 技术关键词
- GitHub 链接
- 是否有文章 / demo / release

### 5. 双身份入口

这是你的站点特色，应该保留。

#### 呈现方式

- 用两个卡片或两个栏目入口
- 不用解释得太学术
- 重点是“同一个人的两种表达”

#### 两个入口建议命名

- `碰个橙子：算法、产品与长思`
- `ChanGe：食物、城市与日常`

### 6. 最近在听

这块很适合你。

#### 原因

- 能把网站从“作品展示”拉回“个人博客”
- 会让整体更柔和
- 很符合你想要的“不那么严肃”

#### 推荐命名

- `最近在听`
- `写东西时常听`
- `散步歌单`
- `今天循环`

### 7. 友链

你提到很喜欢从别人的友链辐射出去，这块非常适合后续保留。

#### 初期做法

- 首页底部放一个“以后会慢慢认识更多有趣站点”的入口
- 或先建一个空的友链页

## 三、栏目 / 分类结构

这部分最关键，因为它决定网站到底像不像“真正博客”。

## 推荐分类

### 1. 长思

放：

- 算法
- 产品
- AI
- 技术观察
- 社会现象与判断

### 2. 项目

放：

- GitHub 项目解读
- 深度学习 / 算法项目记录
- 复盘与实验

### 3. 日常

放：

- 食物
- 城市体验
- 小片段
- 情绪记录

### 4. 读写

放：

- 书影音
- 文章摘录
- 写作过程

### 5. 说说

放：

- 很短的即时表达
- 不适合写成长文的观察

1. 项目名
2. 背景
3. 你做了什么
4. 技术栈
5. 仓库链接
6. 相关文章 / 复盘

### 适合你放进去的重点项目

后续可以继续从 GitHub 中人工补精选项目。

## 3. 文章页

文章页就是标准博客列表页，不用特别魔改。

但建议把置顶内容先配好，让别人一进来就看到你最能代表你的内容。

### 置顶推荐

- 一篇产品拆解
- 一篇 AI / 技术观察
- 一篇更个人化但依然很有判断的文章

## 4. 歌单页

这个页面建议直接单独做。

### 歌单页可以放什么

- 一个外部歌单链接
- 几段简短说明
- 若干首“反复循环”的歌
- 不同时段 / 情绪的歌单

### 页面文案方向

- “歌单是另一种自我介绍。”
- “有些东西不适合写成长文，但适合被听见。”

## 5. 友链页

这个页面很适合后续慢慢长。

### 友链页的意义

- 让你的网站有“在网上真实生活”的感觉
- 呼应你喜欢从友链辐射更多博客这件事
- 会让站点更接近你向往的那种个人博客生态

## 五、侧边栏结构

Butterfly 的侧边栏很适合用来做“轻简历 + 轻名片”。

### 推荐侧边栏内容

- 头像
- 站点名：碰个橙子
- 一句话介绍
- 文章 / 分类 / 标签数量
- GitHub
- 小红书两个入口
- 邮箱
- 最近文章
- 分类
- 标签

### 不建议在侧边栏放太多的内容

- 全量技能
- 过长的自我介绍
- 太像简历的表格信息
